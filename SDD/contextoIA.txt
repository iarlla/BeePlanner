# CONTEXTO DO PROJETO: BEEPLANNER
O BeePlanner é um MVP de um aplicativo de produtividade e engenharia de vida (PWA, acessível via Web e Mobile). Ele monitora metas anuais baseadas na metodologia "Roda da Vida" e exige um cumprimento de 75% para sucesso.

# STACK TECNOLÓGICA
- Frontend/UI: Python com framework Flet (arquitetura reativa, PWA, sem login no MVP).
- Backend/Persistência: Banco de Dados Relacional em nuvem (PostgreSQL/SQLite) com sincronização em tempo real.
- Integrações: Google Tasks (via script) e envio de Ata Diária via SMTP.

# ARQUITETURA DE DADOS (STAR SCHEMA ADAPTADO)
- `configuracao_roda`: Define predefinições (ex: Triângulo, Hexágono).
- `dim_eixos`: Áreas da vida dinâmicas (limite de 3 a 12 eixos), vinculadas à configuração.
- `dim_metas`: Cadastro de hábitos/tarefas, com cálculo de "total_dias_100" e "alvo_75_percent".
- `fact_execucoes`: Tabela fato com os check-ins diários.
- `diario_bordo`: Ata textual qualitativa do dia.

# REGRAS DE NEGÓCIO PRINCIPAIS (RN)
1. Roda da Vida Dinâmica: O gráfico radar no dashboard deve se adaptar matematicamente para renderizar de 3 a 12 eixos usando trigonometria no Flet.
2. Transição "De-Para": Se o usuário reduzir a quantidade de eixos (ex: de Hexágono para Triângulo), o sistema nunca deleta as metas. Ele exige um mapeamento (De-Para) para realocar as metas aos novos eixos (UPDATE via transação SQL).
3. Check-ins: A validação de progresso é calculada via View SQL (`vw_radar_roda_da_vida`).

# DIRETRIZES DE RESPOSTA
- Ao gerar código Flet, priorize componentização e responsividade (mobile-first).
- Ao gerar SQL, garanta transações (BEGIN/COMMIT) para operações críticas de DML.
- Não sugira mudanças de stack. Foque na resolução eficiente dentro do ecossistema Python.
